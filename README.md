# 読書記録アプリ（Reading Log）

## 1. アプリ概要

読みたい本・読書中の本・読了した本を一元管理し、  
読書の進捗・評価・レビューを記録できるアプリ。  
楽天Books API と連携して書籍情報をかんたんに登録でき、  
本棚から詳細への滑らかな Hero 遷移や、月別の読書統計などを通じて、  
**「本を読む体験」そのものを楽しく、継続しやすくする** ことを目的とする。

---

## 2. 企画背景 / 課題

### 現状の課題

- 読みたい本のリストが頭の中だけで管理されている
  - 書店で「あれ何だっけ」となり、結局買えない／買い逃す
  - 友人やSNSで知った本をメモせず忘れてしまう
- 「積読」が増えていくが、一覧で把握できる場所がない
  - 何冊あるのか、どこから読み始めるべきか分からない
- 読んだ本の感想や引用を後から振り返れない
  - ブログを書くほどではないが、自分用のメモは残しておきたい
- 読書のペースが分からず、目標が立てづらい
  - 今月何冊読んだのか、去年と比べてどうか、感覚的にしか把握できない

### 解決策

このアプリでは、本を **「読みたい / 読書中 / 読了」** の 3 ステータスで一元管理し、  
楽天Books API で本を検索 → タップで登録 → 進捗・評価・レビューを記録できるようにする。  
読書記録は蓄積されるほど価値が高まり、月別の統計グラフで読書ペースを可視化することで、  
**続けるほど楽しくなる読書体験** を提供する。

## 2-1. このアプリを選んだ理由

### なぜ「読書記録アプリ」なのか

当初は「クーポン・セール整理帳」を検討していました。  
しかし、企画段階で実装内容と自分の関心を改めて見直した結果、  
以下の理由から **「読書記録アプリ」** に題材を転換することにしました。

#### 1. Flutter らしさを最大限ショーケースできる
- **Hero アニメーション**（本棚 ↔ 詳細）：Flutter ならではの滑らかな画面遷移
- **GridView** で実物の本棚のように表紙を並べる表現
- **CachedNetworkImage** による表紙画像のキャッシュ
- **外部API（楽天Books）連携**：async/await と FutureBuilder の自然な活用
- **Riverpod** によるリアクティブな書籍リスト管理
- **fl_chart** による統計グラフ
- → クーポン整理帳ではこれらをショーケースする機会が限定的だった

#### 2. 実装の難易度が適切
- データモデル → ローカルDB → 外部API → UI → 統計、と段階的に積み上げられる
- 8週間（週6時間、計48時間）の予算でちょうど取り組める規模
- 一つひとつのフェーズが Flutter の主要機能の学習と直結

#### 3. 拡張性が高い
- 積読ビュー、引用集、月間目標通知、読書ペース予測など、自然な機能追加が可能
- データが蓄積するほど価値が高まる仕組み（統計機能で長期的に楽しめる）
- 4年次以降の追加実装にも余地がある

#### 4. 継続して使える実用ツール
- 一過性ではなく、本を読み続ける限り使い続けられる
- 過去の読書履歴がそのまま資産として残る
- 「自分が作ったアプリを自分で使い続ける」というポートフォリオ価値

---

## 3. ターゲットユーザー

- 読書記録を続けたいが、専用ノートを開くほどではないと感じている人
- 「積読」を整理して、計画的に本を読みたい人
- 読んだ本の感想や引用を、自分用にコンパクトに残したい人
- 月にどれくらい本を読んでいるか、データで把握したい人
- スマホで完結する、シンプルな読書ログツールを探している人

### 最低限機能（MVP・3年次・W1〜W4 で実装）

| 機能名             | 内容                                                              |
|--------------------|-------------------------------------------------------------------|
| 本の検索           | 楽天Books API でタイトル・著者から本を検索                        |

| 機能名             | 内容                                                              |
|--------------------|-------------------------------------------------------------------|
| 読書進捗           | 現在のページ数を記録し、進捗バーで可視化                          |
| ★評価             | 5段階の星評価を本ごとに付与                                       |
| レビュー記録       | 本ごとに感想や引用を記録（複数のメモ可）                          |
| ステータスタブ     | 「全て / 読みたい / 読書中 / 読了」のタブで本棚を切り替え         |
| フィルタ・ソート   | 評価順・読了日順・タイトル順での並び替え                          |
| 統計画面           | 月別の完読数・累計冊数・読書ペースをグラフで可視化（fl_chart）    |
| ISBNバーコード読取 | カメラで書籍バーコードをスキャンして自動登録（mobile_scanner）    |
| 月間目標 + 通知    | 「今月◯冊読む」目標と進捗通知（flutter_local_notifications）      |

## 5. 画面構成

本アプリは 5 画面で構成する。

### BookshelfPage（本棚画面・メイン）
- GridView で本の表紙を本棚のように一覧表示
- 上部にタブ：「全て / 読みたい / 読書中 / 読了」
- 右下に FAB（検索画面への遷移ボタン）
- 表紙タップ → BookDetailPage へ **Hero アニメーションで遷移**

### SearchPage（検索画面）
- 検索バーにタイトル・著者を入力
- 楽天Books API を呼び出して結果を一覧表示
- FutureBuilder でローディング状態を管理
- 検索結果をタップ → 登録ダイアログ（ステータスを選んで本棚に追加）

### BookDetailPage（詳細画面）
- 表紙画像（Hero アニメーションで本棚から滑らかに遷移）
- タイトル・著者・出版社・ページ数などのメタ情報
- 現在のステータス変更（読みたい/読書中/読了）
- ★評価（5段階）
- 読書進捗バー（現在のページ / 総ページ）
- レビュー一覧 → タップで ReviewEditPage へ

### ReviewEditPage（レビュー編集画面）
- レビュー本文の入力・編集
- 作成日時の自動記録
- 同じ本に対して複数のレビュー（引用メモ等）を保存可能

### StatsPage（統計画面）
- 月別の完読数（fl_chart の棒グラフ）
- 累計冊数の推移（折れ線グラフ）
- 読書ペース（月平均、ペース予測）

## 6. 技術スタック

| カテゴリ           | 採用                                                              |
|--------------------|-------------------------------------------------------------------|
| フレームワーク     | Flutter（SDK ^3.9.2）                                             |
| 言語               | Dart                                                              |
| 外部API            | 楽天Books API（書籍検索）                                         |
| ローカルDB         | hive（候補：sqflite）                                             |
| 状態管理           | Riverpod（候補：Provider）                                        |
| HTTP通信           | http                                                              |
| 画像キャッシュ     | cached_network_image                                              |
| 統計グラフ         | fl_chart                                                          |
| ★評価UI           | flutter_rating_bar                                                |
| 通知（任意）       | flutter_local_notifications                                       |
| バーコード（任意） | mobile_scanner                                                    |

### Flutter らしさを活かす要素

- **Hero アニメーション**：本棚 → 詳細の表紙遷移（看板機能）
- **GridView**：本棚レイアウト
- **async/await + FutureBuilder**：楽天API 連携時のローディング表示
- **CachedNetworkImage**：表紙画像の効率的なキャッシュ
- **Riverpod**：書籍リストのリアクティブな状態管理

## 8. 今後の拡張アイデア

- **積読ビュー**：未読の本だけを可視化する専用ビュー
- **引用集**：本から印象的な一節を写真と一緒に保存
- **月間目標と進捗通知**：「今月◯冊」目標と達成通知
- **読書ペース予測**：直近の読書ペースから「読了予測日」を算出
- **クラウド同期**：複数デバイス間で読書記録を共有
- **読書時間の記録**：1セッションの読書時間を計測してデータ化
- **タグ機能**：ジャンルや気分のタグで本を分類
